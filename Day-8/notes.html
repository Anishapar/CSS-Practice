Positioning Property

  1. Static
    • Default position: No special positioning. Elements appear in the normal document flow.
    • No effect of top, left, right, bottom.
  2. Relative
    • Stays in document flow: The element occupies its original space.
    • Offset using top, left, right, bottom: Can be moved from its normal position.
    • Reference: The element itself (its original position).
  3. Absolute
    • Leaves document flow: The element doesn't occupy space; other elements ignore it.
    • Offset using top, left, right, bottom: Positions relative to its nearest positioned ancestor.
    • Default reference: The body or the nearest ancestor with relative, absolute, fixed, or sticky positioning.
  4. Fixed
    • Leaves document flow: The element doesn't occupy space in the normal document flow.
    • Fixed to viewport: Stays in the same position relative to the viewport even when scrolling.
    • Offset using top, left, right, bottom: Positions relative to the viewport.
  5. Sticky
    • Hybrid behavior: Acts like relative until it reaches a certain scroll position, then it acts like fixed.
    • Offset using top, left, right, bottom: Sticks to its offset position when scrolling past a specified threshold.
    • Reference: The closest scrolling ancestor.
